# 性能测试

## 负载测试（Load Test）

### 场景设计
典型的负载测试分三个阶段：
1. **加压阶段**（约 30 秒）：逐步增加到目标并发用户数（如 20 用户）
2. **稳定阶段**（约 1 分钟）：保持目标并发用户数
3. **减压阶段**（约 30 秒）：逐步降至 0

### 阈值标准
- 95% 的请求响应时间 < 500ms
- 错误率 < 1%

### 验证要点
- 响应状态码为 200
- 单次请求响应时间 < 200ms

## 压力测试（Stress Test）

### 场景设计
逐步增加负载以找到系统极限：
1. 2 分钟内加压至 100 用户 → 保持 5 分钟
2. 2 分钟内加压至 200 用户 → 保持 5 分钟
3. 2 分钟内降至 0

### 关注要点
- 系统在什么负载水平开始出现性能下降
- 系统崩溃点（最大承载能力）
- 高负载下的错误处理是否正常
- 负载降低后系统能否恢复正常

## 峰值测试（Spike Test）

### 场景设计
模拟突发流量场景：
1. 正常负载（10 用户）运行 1 分钟
2. 10 秒内急剧增加到 200 用户（峰值）
3. 保持高负载 3 分钟
4. 10 秒内降回 10 用户
5. 正常负载运行 3 分钟后结束

### 关注要点
- 峰值期间的响应时间和错误率
- 系统恢复到正常负载后的表现
- 是否有请求丢失或超时

## 浸泡测试（Soak Test）

### 场景说明
在正常负载下长时间运行（数小时至数天），检测：
- 内存泄漏（内存使用是否持续增长）
- 连接池耗尽
- 日志文件增长导致磁盘满
- 性能随时间衰退

## 带认证的 API 性能测试

### 测试策略
1. **准备阶段**：先通过登录接口获取认证 Token
2. **测试阶段**：使用 Token 在请求头中（`Authorization: Bearer {token}`）访问受保护接口
3. 确保 Token 在整个测试期间有效

## 阈值参考

### 响应时间
- P95 < 500ms（95% 的请求在 500ms 内完成）
- P99 < 1000ms（99% 的请求在 1 秒内完成）

### 错误率
- 失败请求率 < 1%

### 吞吐量
- 请求速率 > 100 RPS（每秒请求数）

### 自定义指标
- 可对特定接口设置独立阈值，如登录接口 P95 < 200ms

## 快速参考

| 指标 | 说明 |
|------|------|
| 响应时间 | 从请求发出到收到响应的总耗时 |
| 失败请求率 | 返回错误状态码的请求占比 |
| 请求速率 | 每秒发出的请求数量 |
| P95 | 95% 的请求在此时间内完成 |
| 吞吐量 | 系统每秒处理的请求/事务数 |

| 测试类型 | 目的 |
|----------|------|
| 负载测试 | 验证正常预期负载下的表现 |
| 压力测试 | 找到系统的崩溃点 |
| 峰值测试 | 验证突发流量的处理能力 |
| 浸泡测试 | 检测长时间运行的稳定性 |
