# 安全测试

## 认证测试

### 测试场景
- **无效凭据**：使用错误密码登录应返回 401
- **过期令牌**：使用过期的 Token 访问受保护资源应返回 401
- **篡改令牌**：修改有效 Token 的末尾字符后应返回 401
- **登录速率限制**：连续多次（如 6 次）登录失败后，即使使用正确密码也应返回 429（请求过多）

### 认证安全要点
- 密码应以不可逆哈希（bcrypt/scrypt/argon2）存储
- 错误提示应避免泄露信息（"用户名或密码错误"而非"用户名不存在"）
- 会话令牌应在登录后重新生成
- 密码重置流程需使用有时效、一次性的令牌

## 授权测试

### 测试场景
- **跨用户资源访问（IDOR）**：用户 A 的 Token 访问用户 B 的数据应返回 403
- **普通用户访问管理员路由**：普通用户 Token 调用管理员 API（如删除用户）应返回 403
- **权限提升**：普通用户通过修改请求参数不应获得更高权限
- **URL 直接访问**：未授权用户直接访问受限 URL 应被拦截

### 授权安全要点
- 权限校验必须在后端执行，不能仅依赖前端隐藏
- 修改请求中的用户 ID 不应能访问他人资源
- 角色权限变更后应立即生效

## 输入验证测试

### SQL 注入
- 在搜索框、登录表单、URL 参数中输入 `'; DROP TABLE users; --` 应被拒绝或安全处理
- 所有用户输入应使用参数化查询
- 数据库错误信息不应暴露给用户

### XSS（跨站脚本攻击）
- 输入字段中包含 `<script>alert("xss")</script>` 的内容应被转义或过滤
- 所有用户输入在输出到 HTML 时必须转义
- 富文本编辑器应使用 HTML 标签白名单

### 文件上传
- 应限制允许上传的文件类型
- 恶意文件（如 .exe）应被拒绝
- 文件类型验证应基于文件内容，而非仅扩展名
- 上传的文件应存储在 Web 根目录之外
- 文件名应重新生成以防止路径遍历

### 命令注入
- 用户输入中的 `;`、`|`、`&&`、`` ` `` 等特殊字符应被安全处理
- 用户输入不应直接拼接到系统命令中

## 安全响应头测试

应验证以下响应头已正确设置：
- `X-Content-Type-Options: nosniff` — 防止 MIME 类型嗅探
- `X-Frame-Options: DENY` — 防止点击劫持
- `Strict-Transport-Security` — 强制 HTTPS
- `Content-Security-Policy` — 限制资源加载来源

## 安全测试清单

| 类别 | 测试内容 |
|------|----------|
| **认证** | 无效凭据、令牌过期、令牌篡改 |
| **输入验证** | SQL 注入、XSS、命令注入 |
| **访问控制** | IDOR（不安全的直接对象引用）、权限提升 |
| **速率限制** | 暴力破解防护、API 滥用防护 |
| **安全头** | CSP、HSTS、X-Frame-Options |
| **数据安全** | PII（个人身份信息）泄露、错误信息泄露 |

## 快速参考

| 漏洞类型 | 测试方法 |
|----------|----------|
| SQL 注入 | 在输入中使用 `'; DROP TABLE--` |
| XSS | 在输入中使用 `<script>alert(1)</script>` |
| IDOR | 尝试访问其他用户的资源 |
| CSRF | 检查 Token 是否缺失或无效 |
| 认证绕过 | 缺少认证信息、使用过期令牌 |
